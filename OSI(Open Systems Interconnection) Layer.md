# OSI Layer:
     The OSI model is a 7-layer way to explain how data travels from one computer to another
     Layer 7 – Application
      What you interact with
          -Your browser, app, API
          -Protocols: HTTP, HTTPS, FTP
          -Example: You type google.com
     Layer 6 – Presentation
      How data is formatted
          -Encryption / decryption
          -Compression
          -Example: HTTPS encryption (SSL/TLS)
     Layer 5 – Session
      Manages the conversation
          -Keeps the session open
          -Handles login sessions
          -Example: Staying logged in to a website
     Layer 4 – Transport
      How data is sent reliably
          -TCP (safe)
          -UDP (fast)
          -Uses ports (80, 443)
          -Example: Ensures data arrives correctly
     Layer 3 – Network
      Where to send data
          -IP addresses
          -Routing between networks
          -Example: Finds the path to Google’s server
     Layer 2 – Data Link
      Local delivery
          -MAC addresses
          -Switches
          -Example: Sends data to the correct machine in the local 
     Layer 1 – Physical
      Actual hardware
          -Cables
          -Wi-Fi signal
          -Electricity
    
     7 App: Browser creates HTTP request
     6 Pres: Encrypts data (HTTPS)
     5 Sess: Keeps session alive 
     4 Trans: TCP sends data
     3 Net: IP routes data
     2 Link: MAC address delivery
     1 Phys: Data sent as signals
